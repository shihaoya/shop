> 更新时间：2026-04-15
> 最新进度：第三阶段完成，订单消息模块已实现

## 项目概述
设计一个积分兑换系统，改善运营方和用户之间的积分兑换体验。采用前后端分离架构，支持多租户模式（每个运营方一个租户）。

## 技术栈

### 后端
- Node.js + Express
- MySQL + Sequelize ORM
- JWT 认证
- bcryptjs 密码加密

### 前端
- Vue 3 + Composition API
- Vite 构建工具
- Element Plus UI 组件库
- Pinia 状态管理
- Vue Router
- Axios HTTP 客户端

### 开发工具
- Git 版本控制
- ESLint 代码检查
- Prettier 代码格式化

## 数据库表结构（当前）

1. **users** - 用户表
2. **tenants** - 租户（运营方）表
3. **products** - 商品表
4. **point_transactions** - 积分流水表
5. **user_tenant_relations** - 用户与运营方关系表
6. **operation_logs** - 操作日志表
7. **tenant_audit_history** - 租户审核历史表
8. **orders** - 订单表
9. **messages** - 消息通知表
10. **uploads** - 文件上传记录表

## 注意事项

1. **积分规则由运营方线下维护**，系统只负责记录和查询
2. **收货地址为可选填**，用户可以线下联系运营方
3. **密码丢失需要联系管理员重置**，生成随机密码后线下告知用户
4. **所有操作需要记录日志**，便于审计和追溯
