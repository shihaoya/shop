## 一、数据库扩展

### 1.1 订单表 (orders)
```sql
CREATE TABLE `orders` (
  `id` INT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '订单ID',
  `order_no` VARCHAR(50) NOT NULL UNIQUE COMMENT '订单编号',
  `user_id` INT UNSIGNED NOT NULL COMMENT '用户ID',
  `tenant_id` INT UNSIGNED NOT NULL COMMENT '租户ID',
  `product_id` INT UNSIGNED NOT NULL COMMENT '商品ID',
  `product_name` VARCHAR(100) NOT NULL COMMENT '商品名称（快照）',
  `product_image` VARCHAR(255) DEFAULT NULL COMMENT '商品图片（快照）',
  `points_spent` INT NOT NULL COMMENT '消耗积分',
  `delivery_address` TEXT DEFAULT NULL COMMENT '收货地址',
  `status` ENUM('pending', 'processing', 'completed', 'cancelled') NOT NULL DEFAULT 'pending' COMMENT '订单状态',
  `cancel_reason` TEXT DEFAULT NULL COMMENT '取消原因',
  `is_deleted` TINYINT NOT NULL DEFAULT 0 COMMENT '逻辑删除',
  `deleted_at` DATETIME DEFAULT NULL COMMENT '删除时间',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  FOREIGN KEY (`user_id`) REFERENCES `users`(`id`) ON DELETE CASCADE,
  FOREIGN KEY (`tenant_id`) REFERENCES `tenants`(`id`) ON DELETE CASCADE,
  FOREIGN KEY (`product_id`) REFERENCES `products`(`id`) ON DELETE CASCADE,
  INDEX `idx_order_no` (`order_no`),
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_tenant_id` (`tenant_id`),
  INDEX `idx_status` (`status`),
  INDEX `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表';
```

### 1.2 消息表 (messages)
```sql
CREATE TABLE `messages` (
  `id` BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '消息ID',
  `user_id` INT UNSIGNED NOT NULL COMMENT '接收用户ID',
  `message_type` ENUM('application', 'order', 'points', 'system') NOT NULL COMMENT '消息类型',
  `title` VARCHAR(200) NOT NULL COMMENT '消息标题',
  `content` TEXT NOT NULL COMMENT '消息内容',
  `related_id` INT UNSIGNED DEFAULT NULL COMMENT '关联ID（订单ID、申请ID等）',
  `is_read` TINYINT NOT NULL DEFAULT 0 COMMENT '是否已读：0-未读，1-已读',
  `is_deleted` TINYINT NOT NULL DEFAULT 0 COMMENT '逻辑删除',
  `deleted_at` DATETIME DEFAULT NULL COMMENT '删除时间',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  FOREIGN KEY (`user_id`) REFERENCES `users`(`id`) ON DELETE CASCADE,
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_is_read` (`is_read`),
  INDEX `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='消息表';
```

## 二、后端开发任务

### 2.1 订单管理模块

#### 任务1：用户下单兑换
**接口**：`POST /api/v1/user/orders`

**请求参数**：
```json
{
  "productId": "number (必填)",
  "tenantId": "number (必填)",
  "deliveryAddress": "string (可选)"
}
```

**业务逻辑**：
1. 验证用户是否已加入该租户（status='approved'）
2. 查询商品信息，验证：
   - 商品是否存在且属于该租户
   - 商品是否上架（status='on_shelf'）
   - 库存是否充足（stock > 0）
3. 查询用户积分余额，验证余额充足
4. 开启数据库事务：
   a. 生成订单编号（格式：ORD + 时间戳 + 随机数）
   b. 创建订单记录，status='pending'
   c. 扣减商品库存（stock - 1）
   d. 扣减用户积分
   e. 插入积分流水记录（transaction_type='exchange'）
   f. 创建消息通知运营方（新用户下单）
5. 提交事务
6. 返回订单信息

**响应示例**：
```json
{
  "code": 200,
  "message": "兑换成功",
  "data": {
    "orderId": 1,
    "orderNo": "ORD20260415123456789",
    "pointsSpent": 100,
    "balanceAfter": 900
  }
}
```

**权限要求**：user

**注意事项**：
- 使用数据库行锁防止超卖
- 使用事务保证数据一致性
- 积分扣减和库存扣减必须原子操作

#### 任务2：用户订单列表

**查询参数**：
- `page`: 页码
- `pageSize`: 每页数量

**业务逻辑**：
1. 查询当前用户的订单列表
2. 支持按状态和租户筛选

4. 不包含已删除的订单

**权限要求**：user

**权限要求**：user

#### 任务4：用户取消订单
**接口**：`POST /api/v1/user/orders/:id/cancel`

**请求参数**：
```json
{
  "reason": "string (可选)"
}
```

**业务逻辑**：
1. 验证订单归属
2. 验证订单状态是否为'pending'（只有待处理可取消）
3. 开启数据库事务：
   a. 更新订单状态为'cancelled'
   b. 恢复商品库存（stock + 1）
   c. 退回用户积分
   d. 插入积分流水记录（transaction_type='add', reason='订单取消退回'）
   e. 创建消息通知运营方
4. 提交事务
5. 记录操作日志

## 五、订单状态机

```
pending (待处理)
  ├─→ processing (处理中) [运营方操作]
  │     └─→ completed (已完成) [运营方操作]
  └─→ cancelled (已取消) [用户或运营方操作]
```

**状态流转规则**：
1. 用户下单后，订单状态为`pending`
2. 运营方可将`pending`改为`processing`或`cancelled`
3. 运营方可将`processing`改为`completed`或`cancelled`
4. 用户只能取消`pending`状态的订单
5. `completed`和`cancelled`状态不可再变更

## 六、消息通知场景

### 6.1 触发场景

| 场景 | 接收者 | 消息类型 | 标题示例 |
|------|--------|----------|---------|
| 用户申请加入运营方 | 运营方 | application | 新用户申请加入 |
| 申请审核结果 | 用户 | application | 申请已通过/已拒绝 |
| 用户下单 | 运营方 | order | 收到新订单 |
| 订单状态变更 | 用户 | order | 订单状态已更新 |
| 订单取消 | 运营方+用户 | order | 订单已取消 |
| 积分变动 | 用户 | points | 积分变动通知 |
| 系统通知 | 所有人 | system | 系统维护通知 |

### 6.2 消息模板

**用户申请加入**：
```
标题：新用户申请加入
内容：用户【{username}】申请加入您的运营方，请及时审核。
```

**用户下单**：
```
标题：收到新订单
内容：用户【{username}】兑换了商品【{productName}】，消耗{points}积分。
```

**订单状态变更**：
```
标题：订单状态已更新
内容：您的订单【{orderNo}】状态已更新为【{status}】。
```

**积分变动**：
```
标题：积分变动通知
内容：您的积分{type}了{points}分，当前余额{balance}分。原因：{reason}
```
