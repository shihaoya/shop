## 一、数据库扩展

### 1.1 商品表 (products)
```sql
CREATE TABLE `products` (
  `id` INT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '商品ID',
  `tenant_id` INT UNSIGNED NOT NULL COMMENT '所属租户ID',
  `name` VARCHAR(100) NOT NULL COMMENT '商品名称',
  `description` TEXT DEFAULT NULL COMMENT '商品描述',
  `image_url` VARCHAR(255) DEFAULT NULL COMMENT '商品图片URL',
  `points_required` INT NOT NULL DEFAULT 0 COMMENT '所需积分',
  `stock` INT NOT NULL DEFAULT 0 COMMENT '库存数量',
  `category` VARCHAR(50) DEFAULT NULL COMMENT '商品分类',
  `status` ENUM('on_shelf', 'off_shelf') NOT NULL DEFAULT 'off_shelf' COMMENT '状态',
  `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序权重',
  `is_deleted` TINYINT NOT NULL DEFAULT 0 COMMENT '逻辑删除',
  `deleted_at` DATETIME DEFAULT NULL COMMENT '删除时间',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  FOREIGN KEY (`tenant_id`) REFERENCES `tenants`(`id`) ON DELETE CASCADE,
  INDEX `idx_tenant_id` (`tenant_id`),
  INDEX `idx_status` (`status`),
  INDEX `idx_category` (`category`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品表';
```

### 1.2 积分流水表 (point_transactions)
```sql
CREATE TABLE `point_transactions` (
  `id` BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '流水ID',
  `user_id` INT UNSIGNED NOT NULL COMMENT '用户ID',
  `tenant_id` INT UNSIGNED NOT NULL COMMENT '租户ID',
  `transaction_type` ENUM('add', 'subtract', 'modify', 'exchange') NOT NULL COMMENT '交易类型',
  `points_change` INT NOT NULL COMMENT '积分变动数量（正数表示增加，负数表示减少）',
  `balance_after` INT NOT NULL COMMENT '变动后余额',
  `reason` VARCHAR(255) NOT NULL COMMENT '变动原因',
  `operator_id` INT UNSIGNED DEFAULT NULL COMMENT '操作人ID（运营方或系统）',
  `operator_name` VARCHAR(50) DEFAULT NULL COMMENT '操作人姓名',
  `related_order_id` INT UNSIGNED DEFAULT NULL COMMENT '关联订单ID（兑换时）',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  FOREIGN KEY (`user_id`) REFERENCES `users`(`id`) ON DELETE CASCADE,
  FOREIGN KEY (`tenant_id`) REFERENCES `tenants`(`id`) ON DELETE CASCADE,
  FOREIGN KEY (`operator_id`) REFERENCES `users`(`id`) ON DELETE SET NULL,
  INDEX `idx_user_tenant` (`user_id`, `tenant_id`),
  INDEX `idx_created_at` (`created_at`),
  INDEX `idx_transaction_type` (`transaction_type`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='积分流水表';
```

## 二、后端开发任务

### 2.1 多租户管理模块

#### 任务1：租户信息查询
**接口**：`GET /api/v1/tenants`

**查询参数**：
- `page`: 页码（默认1）
- `pageSize`: 每页数量（默认20）
- `status`: 状态筛选（可选）

**业务逻辑**：
1. 根据当前用户角色返回不同数据
   - admin：返回所有租户
   - operator：返回自己的租户信息
   - user：返回已审核通过的租户列表

**权限要求**：所有登录用户

#### 任务2：租户详情
**接口**：`GET /api/v1/tenants/:id`

**业务逻辑**：
1. 验证租户是否存在
2. 返回租户详细信息

### 2.2 商品管理模块（运营方）

#### 任务3：创建商品
**接口**：`POST /api/v1/operator/products`

**请求参数**：
```json
{
  "name": "string (必填)",
  "description": "string (可选)",
  "imageUrl": "string (可选)",
  "pointsRequired": "number (必填, >0)",
  "stock": "number (必填, >=0)",
  "category": "string (可选)"
}
```

**业务逻辑**：
1. 验证当前用户是否有operator角色
2. 获取用户关联的租户ID
3. 验证参数合法性
4. 创建商品记录，默认下架状态
5. 记录操作日志
6. 返回创建的商品信息

**权限要求**：operator，且只能在自己租户下创建

#### 任务4：商品列表
**接口**：`GET /api/v1/operator/products`

**查询参数**：
- `page`: 页码
- `pageSize`: 每页数量
- `status`: 状态筛选（on_shelf/off_shelf）
- `category`: 分类筛选
- `keyword`: 关键词搜索（商品名称）

**业务逻辑**：
1. 获取当前用户的租户ID
2. 查询该租户下的商品列表
3. 支持分页和筛选
4. 不返回已删除的商品

**权限要求**：operator

#### 任务5：商品详情
**接口**：`GET /api/v1/operator/products/:id`

**业务逻辑**：
1. 验证商品是否存在且属于当前用户的租户
2. 返回商品详细信息

**权限要求**：operator

#### 任务6：更新商品
**接口**：`PUT /api/v1/operator/products/:id`

**请求参数**：同创建商品（所有字段可选）

**业务逻辑**：
1. 验证商品归属
2. 更新商品信息
3. 记录操作日志

**权限要求**：operator

#### 任务7：上架/下架商品
**接口**：`PUT /api/v1/operator/products/:id/status`

**请求参数**：
```json
{
  "status": "on_shelf | off_shelf"
}
```

**业务逻辑**：
1. 验证商品归属
2. 更新商品状态
3. 记录操作日志

**权限要求**：operator

#### 任务8：删除商品（逻辑删除）

**权限要求**：operator

#### 任务9：商品回收站
**接口**：
- `GET /api/v1/operator/products/trash` - 获取已删除商品列表

### 2.3 用户管理模块（运营方）

#### 任务10：添加已注册用户
**接口**：`POST /api/v1/operator/users/add-existing`

**请求参数**：
```json
{
  "username": "string (必填)",
  "initialPoints": "number (必填, >=0)"
}
```

**业务逻辑**：
1. 根据用户名查找用户
2. 验证用户是否存在且未被删除
3. 检查该用户是否已经关联到当前租户
4. 在user_tenant_relations表中创建关联记录
5. 设置初始积分
6. 记录操作日志

**权限要求**：operator

#### 任务11：创建新用户
**接口**：`POST /api/v1/operator/users/create-new`

**请求参数**：
```json
{
  "username": "string (必填, 3-50字符)",
  "password": "string (可选, 不填则使用默认密码)",
  "nickname": "string (可选)",
  "initialPoints": "number (必填, >=0)"
}
```

**业务逻辑**：
1. 验证用户名是否已存在
2. 如果未提供密码，使用默认密码（如：123456）
3. bcrypt加密密码
4. 创建用户记录，role为'user'
5. 创建用户-租户关联记录
6. 设置初始积分
7. 记录操作日志

**权限要求**：operator

#### 任务12：批量导入用户
**接口**：`POST /api/v1/operator/users/import`

**请求参数**：FormData格式，包含CSV/Excel文件

**业务逻辑**：
1. 解析上传的文件
2. 逐行验证数据合法性
3. 对于每个用户：
   - 如果用户名已存在，跳过或更新
   - 如果不存在，创建新用户
   - 创建用户-租户关联
4. 统计成功和失败数量
5. 返回导入结果

**响应示例**：
```json
{
  "code": 200,
  "message": "导入完成",
  "data": {
    "total": 100,
    "success": 95,
    "failed": 5,
    "errors": [
      {
        "row": 10,
        "username": "test",
        "reason": "用户名已存在"
      }
    ]
  }
}
```

**权限要求**：operator

#### 任务13：用户列表
**接口**：`GET /api/v1/operator/users`

**查询参数**：
- `page`: 页码
- `pageSize`: 每页数量
- `keyword`: 用户名搜索

**业务逻辑**：
1. 获取当前用户的租户ID
2. 查询user_tenant_relations表中该租户下的用户
3. 关联users表获取用户基本信息

**权限要求**：operator

#### 任务14：用户详情
**接口**：`GET /api/v1/operator/users/:userId`

**业务逻辑**：
1. 验证用户是否属于当前租户
2. 返回用户详细信息：
   - 基本信息
   - 当前积分余额
   - 历史兑换记录数量
   - 积分流水统计

**权限要求**：operator

#### 任务15：移除用户（逻辑删除）
**接口**：`DELETE /api/v1/operator/users/:userId`

**业务逻辑**：
1. 验证用户是否属于当前租户
2. 在user_tenant_relations表中设置is_deleted=1
3. 记录操作日志
4. 注意：不删除users表中的用户记录

**权限要求**：operator

#### 任务16：恢复用户
**接口**：`POST /api/v1/operator/users/:userId/restore`

**业务逻辑**：
1. 找到已删除的用户-租户关联记录
2. 设置is_deleted=0，deleted_at=NULL
3. 记录操作日志

**权限要求**：operator

#### 任务17：用户回收站
**接口**：`GET /api/v1/operator/users/trash`

**业务逻辑**：
1. 查询is_deleted=1的用户-租户关联记录

**权限要求**：operator

### 2.4 积分管理模块

#### 任务18：用户积分列表
**接口**：`GET /api/v1/operator/points/users`

**查询参数**：
- `page`: 页码
- `pageSize`: 每页数量
- `keyword`: 用户名搜索

**业务逻辑**：
1. 获取当前用户的租户ID
2. 查询该租户下所有用户的积分余额
3. 按用户名搜索
4. 返回用户列表和积分余额

**权限要求**：operator

#### 任务19：增加积分
**接口**：`POST /api/v1/operator/points/:userId/add`

**请求参数**：
```json
{
  "points": "number (必填, >0)",
  "reason": "string (必填)"
}
```

**业务逻辑**：
1. 验证用户是否属于当前租户
2. 验证积分数量为正整数
3. 验证原因不为空
4. 开启数据库事务：
   - 更新user_tenant_relations表的points_balance
   - 插入point_transactions记录（transaction_type='add'）
5. 提交事务
6. 发送积分变动通知给用户
7. 记录操作日志

**权限要求**：operator

#### 任务20：减少积分
**接口**：`POST /api/v1/operator/points/:userId/subtract`

**请求参数**：
```json
{
  "points": "number (必填, >0)",
  "reason": "string (必填)"
}
```

**业务逻辑**：
1. 验证用户是否属于当前租户
2. 验证积分数量为正整数
3. 验证原因不为空
4. 查询当前积分余额
5. 验证余额充足（余额 >= 减少数量）
6. 开启数据库事务：
   - 更新points_balance
   - 插入point_transactions记录（transaction_type='subtract'）
7. 提交事务
8. 发送积分变动通知
9. 记录操作日志

**权限要求**：operator

#### 任务21：修改积分
**接口**：`POST /api/v1/operator/points/:userId/modify`

**请求参数**：
```json
{
  "newPoints": "number (必填, >=0)",
  "reason": "string (必填)"
}
```

**业务逻辑**：
1. 验证用户是否属于当前租户
2. 验证新积分为非负整数
3. 验证原因不为空
4. 查询当前积分余额
5. 计算变动数量 = newPoints - currentBalance
6. 开启数据库事务：
   - 更新points_balance为newPoints
   - 插入point_transactions记录（transaction_type='modify'）
7. 提交事务
8. 发送积分变动通知
9. 记录操作日志

**权限要求**：operator

#### 任务22：批量调整积分
**接口**：`POST /api/v1/operator/points/batch-adjust`

**请求参数**：
```json
{
  "userIds": ["number"] (必填),
  "adjustType": "add | subtract | modify" (必填),
  "points": "number (必填, >0)",
  "reason": "string (必填)"
}
```

**业务逻辑**：
1. 验证所有用户都属于当前租户
2. 验证调整类型和参数
3. 逐个处理每个用户：
   - 根据adjustType执行相应操作
   - 每个用户的操作独立事务
4. 统计成功和失败数量
5. 返回调整结果

**权限要求**：operator

#### 任务23：用户积分流水
**接口**：`GET /api/v1/operator/points/:userId/transactions`

**查询参数**：
- `page`: 页码
- `pageSize`: 每页数量
- `startDate`: 开始日期
- `endDate`: 结束日期
- `transactionType`: 交易类型筛选

**业务逻辑**：
1. 验证用户是否属于当前租户
2. 查询该用户在该租户下的积分流水
3. 支持时间范围和类型筛选
4. 按时间倒序排列

**权限要求**：operator

#### 任务24：导出积分流水
**接口**：`GET /api/v1/operator/points/:userId/transactions/export`

**查询参数**：同任务23

**业务逻辑**：
1. 查询符合条件的流水记录
2. 生成Excel文件
3. 返回文件流下载

**权限要求**：operator

### 7.1 运营方审核状态管理

**功能描述**：运营方可以查看自己的审核状态，被拒绝后可以重新提交申请。

#### 后端接口

**GET /api/v1/tenants/my-status**
- 获取运营方自己的租户审核状态
- 需要 operator 角色认证
- 返回租户详细信息（包括状态、拒绝原因等）

**PUT /api/v1/tenants/resubmit**
- 运营方重新提交审核申请
- 需要 operator 角色认证
- 只有被拒绝的状态才能重新提交

  ```json
  {
    "name": "新的租户名称",
    "description": "新的申请描述"
  }
  ```

#### 前端页面
- **路径**: `/operator/audit-status`
- **组件**: `src/views/Operator/AuditStatus.vue`
- **功能**:
  - 显示当前审核状态（待审核/已通过/已拒绝/已禁用）
  - 被拒绝时显示管理员的拒绝原因
  - 被拒绝时提供“重新提交审核”按钮
  - 重新提交时弹出对话框填写租户名称和申请描述

#### 菜单入口
- 运营方侧边栏新增“审核状态”菜单项（第一个菜单）

### 7.3 租户审核历史记录

**功能描述**：为了实现审核操作的可追溯性，系统新增了租户审核历史记录功能。每次管理员对运营方申请进行审核（通过/拒绝）或修改审核状态时，都会自动创建一条审核历史记录，永久保存而不被覆盖或删除。

#### 数据库设计

**新增表：tenant_audit_history**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT UNSIGNED | 审核历史ID（主键） |
| tenant_id | INT UNSIGNED | 租户ID |
| previous_status | ENUM | 审核前状态（pending/approved/rejected/disabled） |
| new_status | ENUM | 审核后状态 |
| audit_result | ENUM | 审核结果（approved-通过，rejected-拒绝） |
| reject_reason | TEXT | 拒绝原因（仅当拒绝时记录） |
| auditor_id | INT UNSIGNED | 审核人ID（管理员） |
| auditor_username | VARCHAR(50) | 审核人用户名 |
| remark | TEXT | 审核备注 |
| created_at | DATETIME | 审核时间 |
| updated_at | DATETIME | 更新时间 |

**索引**：
- `tenant_id`: 快速查询某个租户的所有审核历史
- `auditor_id`: 查询某个管理员的审核记录
- `audit_result`: 按审核结果筛选
- `created_at`: 按时间排序

#### 后端实现

**模型文件**：`backend/src/models/TenantAuditHistory.js`

**关联关系**：
- Tenant.hasMany(TenantAuditHistory)
- TenantAuditHistory.belongsTo(User, as: 'auditor')

**Service层修改**：`backend/src/services/adminService.js`

1. **approveTenant()** - 通过审核时创建历史记录
2. **rejectTenant()** - 拒绝审核时创建历史记录
3. **updateTenantStatus()** - 手动修改审核状态时创建历史记录

**新增方法**：
```javascript
getTenantAuditHistory(tenantId, page, pageSize)
// 获取指定租户的审核历史列表（分页）
```

**Controller层**：`backend/src/controllers/adminController.js`

新增接口：
```javascript
GET /api/v1/admin/tenants/:tenantId/audit-history
// 获取租户审核历史
```

#### 前端实现

**页面路径**：`/admin/tenant-audit-history/:tenantId`

**功能**：
- 显示租户的基本信息
- 以时间线形式展示所有审核历史
- 每条记录显示：审核时间、审核人、操作类型、状态变化、拒绝原因、备注
- 支持分页

**组件位置**：`src/views/Admin/TenantAuditHistory.vue`
