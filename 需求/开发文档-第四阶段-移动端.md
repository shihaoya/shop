## 一、技术选型

### 1.1 前端技术栈
- **框架**：Vue 3 + Vite（与PC端保持一致）
- **UI组件库**：Vant 4（轻量、可靠的移动端Vue组件库）
- **状态管理**：Pinia
- **路由**：Vue Router
- **HTTP客户端**：Axios
- **适配方案**：viewport单位 + postcss-px-to-viewport
- **打包工具**：Vite

### 1.2 为什么选择Vant
- 专为移动端设计
- 组件丰富且轻量
- 良好的TypeScript支持
- 完善的文档和示例
- 与Vue 3完美集成

## 三、移动端设计规范

### 3.1 设计原则
1. **简洁直观**：减少操作步骤，一目了然
2. **触摸友好**：按钮和点击区域足够大（最小44x44px）
3. **加载快速**：首屏加载时间控制在2秒内
4. **反馈及时**：操作后立即给予视觉反馈
5. **容错性强**：提供清晰的错误提示和恢复方式

### 3.2 布局规范
- **安全区域**：考虑刘海屏、底部横条等
- **字体大小**：正文14-16px，标题16-18px
- **间距规范**：使用8px网格系统
- **颜色规范**：
  - 主色：#1989fa（蓝色）
  - 成功：#07c160（绿色）
  - 警告：#ff976a（橙色）
  - 危险：#ee0a24（红色）
  - 文字：#323233（深灰）、#969799（浅灰）

### 3.3 交互规范
- **下拉刷新**：列表页面支持下拉刷新
- **上拉加载**：列表页面支持上拉加载更多
- **左滑操作**：列表项支持左滑显示操作按钮
- **长按操作**：某些场景支持长按显示菜单
- **手势返回**：支持边缘右滑返回上一页

## 四、后端适配（可选）

### 4.1 User-Agent识别
在后端添加中间件识别移动端请求：

```javascript
// middleware/device.js
function detectDevice(req, res, next) {
  const userAgent = req.headers['user-agent'] || '';
  const isMobile = /Mobile|Android|iPhone|iPad/i.test(userAgent);
  req.isMobile = isMobile;
  next();
}
```

### 4.2 API兼容性
- 移动端和PC端共用同一套API
- 如需差异化响应，可通过User-Agent或请求参数区分
- 建议保持API一致性，减少维护成本

## 八、部署说明

### 8.1 构建命令
```bash
# 开发环境
npm run dev

# 生产环境构建
npm run build
```

### 8.2 部署方式
- **静态托管**：Nginx、Apache
- **CDN部署**：阿里云OSS、腾讯云COS
- **服务器部署**：Node.js静态文件服务

### 8.3 Nginx配置示例
```nginx
server {
    listen 80;
    server_name m.example.com;
    root /var/www/mobile;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api/ {
        proxy_pass http://backend-server;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
