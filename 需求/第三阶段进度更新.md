## 🔧 技术要点

### 消息系统设计
- 采用异步发送，不阻塞主业务流程
- 使用 `Promise.allSettled` 批量发送，部分失败不影响其他消息
- 错误捕获和日志记录，避免消息发送失败影响核心业务

### 订单事务处理
- 使用 Sequelize 事务保证数据一致性
- 积分扣减、库存扣减、订单创建、流水记录原子操作
- 异常情况自动回滚

### 消息通知时机
- 订单创建 → 通知运营方
- 订单状态变更 → 通知用户
- 订单取消 → 通知双方
- 积分变动 → 通知用户
- 申请审核 → 通知双方
